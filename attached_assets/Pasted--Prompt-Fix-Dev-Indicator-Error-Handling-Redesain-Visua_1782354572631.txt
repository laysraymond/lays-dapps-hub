## Part 1 — Disable the Next.js dev indicator (bottom-left icon/menu)

The icon and popup menu showing "Route: Static", "Try Turbopack", "Route Info", "Preferences" in the bottom-left corner is Next.js's built-in development indicator, not part of our app code. Disable it in `next.config.js` / `next.config.ts`:

```ts
const nextConfig = {
  devIndicators: false,
}
```

(If this project's Next.js version uses a different shape for this option, check the installed Next.js version's docs for the correct `devIndicators` syntax and apply the equivalent — the goal is simply: no dev indicator badge/menu visible in the bottom-left corner.)

## Part 2 — Handle the wallet "not authorized" error gracefully

The runtime error "The source https://[replit-preview-url] has not been authorized yet" is thrown by the browser's wallet extension itself (visible in the stack trace as a `chrome-extension://` source) — it happens when the wallet extension hasn't granted this specific origin permission yet, or when a previously-connected session becomes stale (e.g. after the Replit preview URL changes on restart). This is not fixable purely in our code (it's the extension's own permission system), but right now it's surfacing as an unhandled runtime error that crashes into Next.js's red error overlay, which looks broken to visitors. Fix the error handling so it degrades gracefully instead:

- Wrap all wallet interactions (`connect()`, `sendTransaction()`, `switchChain()`, etc.) in try/catch blocks.
- When a call fails with an authorization/permission-related error (check the error message/code for things like "not been authorized", "not authorized by the user", EIP-1193 error code `4100`, or a rejected/disconnected provider state), do NOT let it bubble up as an unhandled exception. Instead, reset the wallet connection state in the UI and show a clear message: "Your wallet connection needs to be refreshed — please reconnect." with a prominent "Reconnect" button that re-triggers the connect flow from scratch.
- This applies across all wallet-gated pages (home, leaderboard, submit, proof-log) and the admin page's wallet check — anywhere a wallet call happens, it should fail soft into a "Reconnect" prompt, never into a raw crash overlay.
